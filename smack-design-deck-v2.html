Section 01

Icon System

Replace emoji chaos with a consistent SVG icon set. 1.5px stroke, rounded caps, 20px base size — Feather-style, production-ready. Every icon below is inline SVG. Zero external dependencies.

The problem: The current app uses 📊📅👥💰 as UI elements. Emoji render inconsistently across OS, can't be tinted for state (active/inactive/disabled), and break in accessibility contexts. This icon set replaces all of them with predictable, styleable SVG.

Section 02

Status System

Every state, precisely communicated. Each entity in Smack has its own status vocabulary. Badges use a dot + label for most contexts, icon + label for high-priority states (overdue, conflict). Zero emoji.

Section 07

Build Notes

What to ship first, in order. Get Direction 1 to pixel-perfect, then layer in the other directions as theme tokens.

Phase 1: Foundation (do this first)
1
Replace all emoji with SVG icons

Find all instances of 📊📅👥💰⚠️✅ etc. in the codebase. Replace with the icon components defined in Section 1. This is a pure visual improvement — no behavior changes.

2
Implement the status badge component

Build a single StatusBadge component that accepts a status string and renders the correct dot color, icon, and label text. Map all 4 entity types: Project / Invoice / Crew / Deliverable.

3
Rebuild the sidebar

Implement the dark navy sidebar with the jelly mark, icon+label items, active/inactive states (salmon vs. gray), and the bottom user area. This is the visual frame for the entire app.

4
Calendar redesign — the big one

Build the week view with all-day shoot bars, time grid, today column tint, and Crew View toggle. The crew availability matrix is the second tab within the same view. This is the screen users will live in.

Phase 2: Core Screens + Phase 3: Polish
5
Dashboard + Projects list

4 metric cards, projects table with budget bars, activity feed with icon-typed items. Filter chips on projects list. Both screens per Section 3.1 and 3.2.

6
Project detail — all tabs

Phase pipeline, tabs (Overview/Crew/Finance/Schedule/Files/Activity), crew table with portal access toggles. The Crew tab is most critical. Budget bar segments in the header.

7
Invoices + Invoice editor

List view with summary stat bar. Two-panel editor with WYSIWYG invoice document on the left. "Overdue" rows get red row highlight. Follow-up CTA button for overdue.

8
Dark mode + multi-theme tokens

Once Direction 1 is solid, extract CSS custom properties to a theme file. Dark mode becomes a body class toggle. Direction 2 is essentially the same components with different token values. Ships as a free user preference.

Key CSS Token Decisions
--salmon: #e07868 — Primary CTA, active sidebar, salmon-tinted metric card
--navy: #0f1d30 — Sidebar background, today's date circle, active phase pill
--green: #059669 — Paid invoices, confirmed crew, complete projects, budget paid bars
--amber: #d97706 — On hold, in review, awaiting — anything "soft pending"
--red: #dc2626 — Overdue, conflict, cancelled — anything that needs action NOW
--blue: #2563eb — Planning, sent, viewed — things that are in motion but not yet resolved
