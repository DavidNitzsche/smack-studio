01 — Current State

WorkprintProjects: honest audit

This app works. It's built real data around it. But it's carrying the weight of iteration after iteration without a design system holding it together. Here's what we see.

Admin App — projects.workprint.la
✓
Keep — Dark navy sidebar with section groups (CREW / MONEY / SYSTEM). Functional, feels premium. Anchor this in Smack.
✓
Keep — Project peek panel (click row → right drawer). Elegant interaction pattern. One of the best things in the app.
✓
Keep — Budget/Spent/Remaining always visible at the top of each project. Financial clarity baked into the context.
✓
Keep — Phase pipeline (Bid → Upcoming → Production → Finishing → Delivered). The concept is right, just needs visual refinement.
!
Fix — Brand inconsistency. Blues everywhere, yellow/blue WP logo, no salmon. Smack needs its own color language applied consistently.
!
Fix — Type hierarchy is ad hoc. "All Projects", "Reports & Analytics", "Bookings" — every page title is styled differently. Needs a system.
!
Fix — Empty states are bare. "No bookings found" with a tiny icon. Empty pages need purpose — explain what will be here, give a clear first action.
!
Fix — Project list cards are thin. One project per card, showing almost nothing. Should surface client, dates, progress bar, crew count at a glance.
!
Fix — Reports has a dark header that breaks the visual rhythm of the whole app. One-off styling that creates inconsistency.
~
Rethink — Schedule / Gantt is sparse. One project, three phases. The concept is correct but the view needs density — crew bookings overlaid, milestone markers, date labels.
~
Rethink — Settings page. A single "Company Name" field with 9 tabs. Information architecture problem — some tabs have a lot, some have almost nothing.
Contractor Portal — portal.workprint.la
✓
Keep — Login page vibe — full-bleed dark blue, centered card, clean. The brand direction is right. Just needs Smack identity swapped in.
✓
Keep — "How it works" 3-step on the home screen for new contractors. Smart onboarding moment. Disappears once they have bookings.
!
Fix — The mascot character (cartoon orange cube with sunglasses) on the login page. Jarring tone mismatch with a professional studio management platform.
!
Fix — "Getting Paid" needs more weight. That's the whole reason contractors are here. It's buried in the sidebar with no visual priority.
!
Fix — Invoice status is unclear. Paid / Awaiting Payment / Pending Review — the user sees three $0.00 cards. The empty state tells them nothing about what to do next.
!
Fix — Portal doesn't feel like Smack. It's WP-branded. When studios start using Smack, their contractors should see Smack's identity — including the studio's name up top.
~
Rethink — Navigation labels. "Time & Pay" is unclear. Does that mean I log time here? Submit for payment? "Log Time" and "My Payments" are more direct.
~
Rethink — Profile page feels like a form, not a workspace. Contractors should see their rates, availability, and booking history at a glance — not just a contact info form.
The core is solid. The craft layer is missing.
The data model is right, the workflows are correct, and the fundamental UX patterns work. What's missing is a consistent design language — spacing, color, type, component standards — applied with intention to every pixel. That's what Smack fixes.
02 — Competition

What the best in the space do well

We're not copying anyone. But we're absolutely stealing their best ideas.

🍯
HoneyBook
Client experience platform — $19-$79/mo
What they nail
  • Pipeline view — Kanban board showing every client's status (Inquiry → Proposal → Contract → Invoice → Complete). Visual clarity that makes the business feel manageable at a glance.
  • Client portal — Feels like a luxury experience. The client sees a beautiful, branded page to sign contracts, pay invoices, and message you. Not a raw link to a form.
  • Smart Files — Proposals, contracts, and invoices all in one linked document. Sign here, pay here, done. The flow is seamless.
  • Onboarding — Guided setup with progress tracking. You feel accomplished in 20 minutes. Completion rate is extremely high.
Steal this: The pipeline view as a top-level navigation concept. Studios should see every active job's status at a glance from the dashboard.
⚙️
Dubsado
CRM & automation for creatives — $20-$40/mo
What they nail
  • Workflow automation — Visual workflow builder where you set triggers and actions. "When contract is signed → send invoice → wait 3 days → send reminder." No code needed.
  • Form builder — Lead capture, questionnaires, and onboarding forms all built in. Conditional logic, file uploads, custom branding.
  • Canned responses — Email templates accessible directly from the message composer. One click to use, easy to personalize.
  • Scheduler — Booking calendar with real-time availability. Clients book directly into your calendar from a public link.
Steal this: The visual workflow concept. Smack should have automations — e.g. "When project moves to Production → auto-remind unpaid invoices."
🌿
Bonsai
Freelancer contracts & invoicing — $25-$55/mo
What they nail
  • Contract signing flow — The most frictionless signing experience in the space. Clean, legal, fast. Contractors see exactly what they're signing. Digital signature in two clicks.
  • Time tracking UX — Simple timer, project tagging, automatic invoice generation from logged hours. The math is always right.
  • Invoice design — Invoices look like designed documents, not database exports. Professional appearance builds client trust.
  • Minimal navigation — Everything accessible in 6 top-level items. No cognitive overload. Clean and fast.
Steal this: The contractor signing experience. When Smack sends a contractor a booking, the acceptance flow needs to be this clean.
◈
Linear
UI reference only — the B2B design gold standard
What they nail
  • 8pt spacing grid applied perfectly — Every element sits on the grid. The visual rhythm is flawless. Nothing feels out of place.
  • Data density without overwhelm — Tables show a lot of information but feel scannable. Font size, weight, color, and spacing all work together.
  • Command palette (⌘K) — Universal search and actions. Keyboard-first design. Power users can do anything without a mouse.
  • Status system — Every issue/project has a clear status with a specific icon + color. The system is learnable in 5 minutes and never ambiguous.
Steal this: The spacing discipline and status system design. Smack's status badges (Active, Booked, Pending, Paid) need the same rigour.
▶
Frame.io
Creative workflow — same industry, different problem
What they nail
  • Dark UI done right — Muted background, tight contrast hierarchy, careful use of color for meaning. Feels like a professional tool, not a consumer app.
  • Contextual actions — The right buttons appear exactly when you need them. Review approved? The approval button appears. Not before.
  • Status pipeline on projects — Every deliverable has a visual status that matches where it is in the workflow. Review, Approved, Sent for Approval, etc.
  • Activity feed — Every action logged, timestamped, attributed. Full audit trail visible in context. Trust-building for clients.
Steal this: The activity feed concept for projects. Every booking, invoice action, and status change should be visible in a project timeline.
🪼
What Smack has that none of them do
Our actual competitive advantages
Our moat
  • Contractor payroll built in — None of the above pay contractors. They track invoices. We process ACH payments, track 1099s, handle the full payroll cycle.
  • Per-project P&L at the shoot level — Budget, crew costs, margin, production fee — all live per job. No one else surfaces this for studio owners.
  • Crew availability across projects — Who's available Apr 15? Which editor is already booked? Cross-project availability is unique to our model.
  • Studio multi-tenancy — When we go to market, one Smack account can run multiple studios. None of the above are built for this.

App Design Rules

Do
  • Use salmon for the ONE primary action per screen
  • Use navy for hero/header backgrounds
  • Use green (#059669) only for money-positive values
  • Show financial values with two decimal places always
  • Use 8pt grid for all spacing — no arbitrary values
  • All page titles: 36px 800 weight, consistent treatment
  • Every empty state has an action button and 1-line explanation
  • Status badges always have a dot + label, never label alone
  • Tables: left-align text, right-align numbers
  • Keyboard shortcut for every frequent action (⌘K for search)
Don't
  • Don't use blue as a primary action color — that's WP, not Smack
  • Don't mix multiple font weights in the same line
  • Don't use red for anything that isn't an error or overdue state
  • Don't put icons without labels in navigation (small screen risk)
  • Don't show empty tables — collapse the section or show a state
  • Don't use more than 3 levels of visual hierarchy per screen
  • Don't break the grid for "design moments" — rhythm beats decoration
  • Don't truncate dollar amounts with "k" in financial detail views
  • Don't put destructive actions (delete) in primary button position
  • Don't auto-refresh data in the background without a visible indicator

Changes: merged Invoices In/Out into one view (tabs within), removed Availability (→ inside Contractors), removed Cash Flow (→ inside Reports). 5 items fewer = cleaner mental model.
The Smack navigation principle
Top-level nav items should be nouns (things), not actions. You navigate to a place, then take actions within it. "Availability" is a property of Contractors — not a destination. "Cash Flow" is a view within Reports — not a separate place.
07 — Build Plan

What we build and in what order

Design before code. Components before screens. Shell before features. This order matters.

1
Design Foundation
Before any screen is coded
→
Finalize all CSS tokens — color vars, spacing scale, radius, shadow, type scale. One source of truth, used everywhere.
→
Build component file — button, input, badge, card, table, empty state, modal. All states (default, hover, focus, disabled, error).
→
Shell layout — sidebar nav, top bar, main content area, right panel drawer. Responsive. Keyboard-navigable.
2
Core Screens
The daily-use surfaces
→
Dashboard — Navy hero, 4 metrics, active projects list, action items (overdue invoices surface here).
→
Projects list — Richer project cards (client, dates, budget bar, crew count, status badge). Peek panel preserved.
→
Project detail — Refined header, pipeline, consolidated tabs. Activity feed as new tab.
→
Today view — Action items, crew on-call today, upcoming deadlines. Should feel like a morning briefing.
3
Money Screens
The revenue engine
→
Invoices — Unified view (In + Out as tabs). Outstanding at top, table below. Invoice detail is a designed document, not a raw form.
→
Payroll — Ready to Pay is the hero. 1099 tracking. Payout history with search/filter. ACH status pipeline.
→
Reports — Drop the dark header. Same canvas as rest of app. Revenue, Projects, Contractors, Cash Flow as tab views.
→
Bids — Pipeline kanban as default view. List toggle. Bid detail page that generates into a project on approval.
4
Crew Screens
The people layer
→
Contractors — Card grid (preferred) + table (roster). Contractor profile page with rates, history, portfolio link, booking list.
→
Bookings — Calendar-first. Week/month/day views. Booking detail as right panel. Color-coded by project.
→
Schedule — Gantt with crew overlay. Drag to reschedule. Milestone markers. Today line. Dense and readable.
5
Contractor Portal Rebuild
The other half of the product
→
Login + auth — Smack-branded (studio name shown). Clean card. No mascot. Magic link or password.
→
Home — Greeting, next booking, payment status (big and bold), outstanding action if any.
→
My Payments — Payment history with status. "Getting Paid" onboarding is a highlighted callout, not buried.
→
Log Time — Booking-linked time entry. Day picker. Submit generates the invoice automatically.
